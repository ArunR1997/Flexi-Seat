Web Services Overview

The web services listed below have been designed using the Representational State Transfer (REST) method for web services. This RESTful method provides a simple and clean interface for interfacing with a backend application.

1. Calling the Web Service

To call the CCS web service, the application initiates an HTTPS Get request to the specified Uniform Resource Locator. The URL will contain the address to the web service, along with any input parameters that are required. This is the basic structure of the URL;

https://<hostname>:<port>/ccs/services/<servicename>/<output>/<appid>?<query parameters>

Notes: The query parameters use the standard name-value pair (e.g. CnyCd=CA) and are appended to the end of the URL string. Order of parameters does not matter. Case of the parameter keys (e.g. CnyCd) does not matter.

CCS Web Services are not intended to be consumed through Internet Explorer or any web browser.
‎ Web browsers may not have enough internal memory to hold the resulting data stream. This can cause truncation in the xml and errors when the web browser attempts to process the results. It can also cause the web browser to stop responding.

Deployment Topology

CCS will run as a Windows .Net Service, using .Net Remoting via Windows Communication Foundation (WCF) and will use Microsoft SQL Server as a data storage facility. CCS components will be living and executing on a dedicated LOE Server in the datacenter. OCCG team will package the installation and will be responsible for installing the CCS.

Transaction Management

The successful transfer of the data will indicate the conclusion of a data transfer transaction. Failed transfers will be retried until successful.

Error Handling

OCCG will report any error if unable to build Committed Lane Data

CCS will return error codes back to calling application as documented in the above section .

Restart/Recovery

CCS will support and handle all SQL errors including reaching out to DDS for its support. If there is any irrecoverable error from SQL, CCS will work with DDS to handle SQL data recovery process.
